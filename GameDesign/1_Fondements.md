# 1. Les fondements du Game Design

## Objectifs pédagogiques

À la fin de ce chapitre, vous serez capable de :

- Comprendre ce qu'est réellement le game design.
- Différencier le game design des autres métiers du jeu vidéo.
- Identifier les composantes fondamentales d'un jeu.
- Comprendre pourquoi certains jeux sont amusants tandis que d'autres échouent.
- Utiliser le vocabulaire de base du game design.

---

# 1.1 Qu'est-ce que le Game Design ?

Le game design est la discipline qui consiste à concevoir les règles, les systèmes et les interactions d'un jeu afin de produire une expérience spécifique chez le joueur.

Contrairement à une idée répandue, le game designer ne dessine pas forcément les personnages, ne programme pas nécessairement le jeu et n'écrit pas toujours le scénario.

Son rôle est de répondre à une question simple :

> "Que va faire le joueur pendant tout le jeu ?"

Le game designer imagine :

- les actions disponibles,
- les objectifs,
- les obstacles,
- les récompenses,
- la progression,
- les choix possibles,
- les conséquences des actions.

En résumé :

Le game designer construit l'expérience du joueur.

---

# 1.2 Définition d'un jeu

Le chercheur Jesper Juul définit un jeu comme un système basé sur des règles où le joueur poursuit un objectif tout en recevant des conséquences mesurables selon ses actions.

Un jeu possède généralement :

- des règles,
- un objectif,
- des contraintes,
- un système de récompense,
- des choix,
- une possibilité d'échec.

Sans règles, il n'y a pas de jeu.

Sans objectif, il n'y a pas de motivation.

Sans choix, il n'y a pas d'interactivité.

---

# 1.3 Les quatre piliers du Game Design

Tous les jeux, quels que soient leur genre ou leur plateforme, reposent sur quatre éléments fondamentaux.

## Les règles

Les règles définissent ce qui est autorisé.

Exemples :

Dans un jeu de plateforme :

- le joueur peut sauter ;
- il ne peut pas traverser les murs ;
- il meurt en tombant dans le vide.

Les règles créent le cadre de l'expérience.

---

## Les objectifs

Les objectifs donnent une direction au joueur.

Ils peuvent être :

- sauver un personnage ;
- atteindre la sortie ;
- battre un boss ;
- survivre dix minutes ;
- construire une ville.

Sans objectif, le joueur perd rapidement sa motivation.

---

## Les choix

Le plaisir naît souvent de la prise de décision.

Un bon jeu présente régulièrement des choix intéressants.

Par exemple :

- Dois-je combattre ou fuir ?
- Dépenser mon argent maintenant ou plus tard ?
- Explorer ou suivre la quête principale ?
- Utiliser une potion ou la conserver ?

Plus un choix possède des avantages et des inconvénients équilibrés, plus il est intéressant.

---

## Le feedback

Chaque action du joueur doit produire une réponse.

Le feedback peut être :

- visuel ;
- sonore ;
- physique (vibration) ;
- numérique (score, dégâts, expérience).

Exemple :

Lorsque l'on frappe un ennemi :

- un son est joué ;
- une animation apparaît ;
- une barre de vie diminue ;
- un nombre de dégâts est affiché.

Le joueur comprend immédiatement l'effet de son action.

---

# 1.4 Les composants d'un jeu

Un jeu est composé de plusieurs couches.

## Les mécaniques

Les mécaniques correspondent aux actions possibles.

Exemples :

- courir ;
- tirer ;
- construire ;
- récolter ;
- négocier ;
- cuisiner.

---

## Les dynamiques

Les dynamiques sont les comportements qui émergent des mécaniques.

Exemple :

Si un jeu permet :

- de construire,
- de détruire,
- de voler,

les joueurs peuvent créer des alliances ou des guerres, même si cela n'était pas explicitement prévu.

---

## L'esthétique

L'esthétique représente l'émotion ressentie.

Un même système de règles peut produire :

- du stress,
- de la joie,
- de la peur,
- de la curiosité,
- de la compétition,
- de la détente.

Le but du game designer est souvent de produire une émotion précise.

---

# 1.5 Les différents métiers

Le développement d'un jeu vidéo rassemble de nombreux spécialistes.

## Game Designer

Conçoit les règles et les systèmes.

---

## Level Designer

Conçoit les niveaux.

Il décide :

- où placer les ennemis ;
- où placer les récompenses ;
- comment guider le joueur.

---

## Narrative Designer

Conçoit l'histoire interactive.

Il écrit :

- les quêtes ;
- les dialogues ;
- les personnages ;
- les embranchements narratifs.

---

## Technical Designer

Fait le lien entre le design et la programmation.

Il crée souvent les systèmes dans des outils visuels comme Blueprint (Unreal Engine).

---

## UX Designer

Travaille sur l'expérience utilisateur.

Il optimise :

- les menus ;
- les tutoriels ;
- l'accessibilité ;
- les interfaces.

---

# 1.6 Les types de joueurs

Tous les joueurs ne recherchent pas la même expérience.

Voici une classification classique.

### Les explorateurs

Ils aiment découvrir.

Ils passent du temps à chercher les secrets.

---

### Les compétiteurs

Ils veulent gagner.

Ils recherchent la maîtrise.

---

### Les collectionneurs

Ils veulent tout obtenir.

Ils adorent les succès et les objets rares.

---

### Les créateurs

Ils aiment construire.

Ils apprécient les jeux offrant une grande liberté.

---

### Les joueurs sociaux

Ils jouent principalement avec d'autres personnes.

Ils privilégient les interactions humaines.

---

# 1.7 Pourquoi un jeu est-il amusant ?

Le plaisir provient souvent d'un équilibre entre :

- difficulté ;
- progression ;
- découverte ;
- maîtrise ;
- récompense.

Un jeu trop simple devient ennuyeux.

Un jeu trop difficile devient frustrant.

Le game designer cherche constamment le juste équilibre.

---

# 1.8 Étude de cas

Prenons un jeu de plateforme classique.

Le joueur peut :

- courir ;
- sauter ;
- éviter des obstacles.

L'objectif est d'atteindre la fin du niveau.

Chaque saut représente une prise de décision.

Le feedback est immédiat :

- animation ;
- musique ;
- score ;
- effets visuels.

Ce système très simple suffit à produire une expérience riche.

# Exercices

## Exercice 1

Choisissez votre jeu préféré.

Identifiez :

- les règles ;
- les objectifs ;
- les choix proposés ;
- les feedbacks.

---

## Exercice 2

Inventez un jeu en une phrase.

Exemple :

> "Un astronaute doit réparer une station spatiale avant que l'oxygène ne soit épuisé."

Listez ensuite :

- les actions possibles ;
- les obstacles ;
- les récompenses.

---

## Exercice 3

Analyse critique

Pourquoi un jeu auquel vous avez joué récemment vous a-t-il donné envie de continuer ?

Essayez d'identifier les mécanismes psychologiques utilisés.

---

# À retenir

- Le game design consiste à créer une expérience.
- Les règles définissent le jeu.
- Les choix créent l'intérêt.
- Les feedbacks informent le joueur.
- Les objectifs motivent la progression.
- Le plaisir provient de l'équilibre entre difficulté, récompense et apprentissage.
