# 5. Le Level Design

## Objectifs pédagogiques

À la fin de ce chapitre, vous serez capable de :

- Comprendre le rôle du Level Design.
- Différencier le Level Design de l'Environment Art.
- Concevoir un niveau qui enseigne sans tutoriel.
- Guider naturellement le joueur.
- Contrôler le rythme d'un niveau.
- Utiliser les notions de lisibilité, de composition et de pacing.
- Créer un premier niveau sur papier.

---

# 5.1 Qu'est-ce que le Level Design ?

Le **Level Design** est la discipline qui consiste à concevoir les espaces dans lesquels évolue le joueur.

Le level designer ne crée pas uniquement un décor.

Il crée une **expérience**.

Son objectif est de répondre à plusieurs questions :

- Où va le joueur ?
- Que découvre-t-il ?
- Quels choix peut-il faire ?
- Quand combat-il ?
- Quand se repose-t-il ?
- Comment apprend-il une nouvelle mécanique ?

Le niveau devient un outil de narration, de gameplay et de progression.

---

# 5.2 Level Design vs Environment Art

Ces deux métiers sont souvent confondus.

## Level Designer

Il construit :

- la structure du niveau ;
- les chemins ;
- les obstacles ;
- les ennemis ;
- les objets ;
- les énigmes ;
- le rythme.

Il travaille principalement avec des formes simples (greybox).

---

## Environment Artist

Il transforme cette structure en environnement crédible.

Il ajoute :

- textures ;
- végétation ;
- bâtiments ;
- éclairages ;
- effets visuels ;
- détails décoratifs.

En résumé :

Le Level Designer crée le **fonctionnement**.

L'Environment Artist crée le **visuel**.

---

# 5.3 Les objectifs d'un niveau

Un niveau doit toujours avoir un but.

Exemples :

- apprendre une mécanique ;
- raconter une histoire ;
- présenter un boss ;
- introduire une nouvelle zone ;
- offrir un défi particulier.

Si un niveau n'apporte rien de nouveau, sa présence doit être remise en question.

---

# 5.4 Les trois questions fondamentales

Le joueur se pose constamment trois questions.

## Où suis-je ?

Le niveau doit être lisible.

Le joueur doit comprendre :

- son environnement ;
- les zones accessibles ;
- les dangers.

---

## Où dois-je aller ?

Le jeu doit guider le joueur.

Sans forcément utiliser une flèche.

---

## Pourquoi y aller ?

Le joueur doit avoir une motivation.

Exemples :

- une récompense ;
- un objectif ;
- une curiosité ;
- un danger à éviter.

---

# 5.5 La lisibilité

La lisibilité désigne la facilité avec laquelle le joueur comprend ce qu'il voit.

Un bon niveau est immédiatement compréhensible.

Le joueur distingue facilement :

- les plateformes ;
- les ennemis ;
- les objets interactifs ;
- les zones dangereuses.

---

## Les contrastes

Le regard est attiré par :

- la lumière ;
- la couleur ;
- le mouvement ;
- la taille.

Le concepteur peut utiliser ces éléments pour orienter naturellement le joueur.

---

## Les silhouettes

Les objets importants doivent avoir une silhouette identifiable.

Même sans texture, le joueur doit reconnaître :

- une caisse ;
- une porte ;
- un ennemi ;
- une échelle.

C'est pourquoi les premiers prototypes sont souvent réalisés sans graphismes détaillés.

---

# 5.6 Le guidage du joueur

Un bon niveau guide le joueur sans lui donner constamment des instructions.

On parle de **guidage implicite**.

## La lumière

L'œil est naturellement attiré par les zones lumineuses.

Une sortie éclairée est plus visible qu'un couloir sombre.

---

## Les lignes

Les routes, les murs ou les rivières peuvent orienter le regard.

Le joueur suit instinctivement ces lignes.

---

## Les repères visuels (Landmarks)

Un repère est un élément facilement reconnaissable.

Exemples :

- une tour ;
- une montagne ;
- un arbre géant ;
- un phare.

Ils permettent au joueur de s'orienter.

---

## Les mouvements

Un objet en mouvement attire immédiatement l'attention.

Exemples :

- une cascade ;
- un moulin ;
- un ennemi ;
- un drapeau.

---

# 5.7 Le rythme (Pacing)

Le rythme correspond à l'alternance entre différentes intensités de jeu.

Un bon niveau alterne généralement :

- exploration ;
- combat ;
- résolution d'énigmes ;
- découverte ;
- repos.

Cette alternance évite la fatigue.

---

## Exemple de séquence

Exploration

↓

Petit combat

↓

Récompense

↓

Puzzle

↓

Découverte d'un panorama

↓

Boss

↓

Cinématique

↓

Zone calme

Cette structure crée une expérience variée.

---

# 5.8 Enseigner sans tutoriel

Les meilleurs jeux enseignent leurs mécaniques directement par le level design.

On parle souvent de **tutorialisation implicite**.

Exemple classique :

Le joueur rencontre un obstacle simple.

↓

Il expérimente.

↓

Il comprend la mécanique.

↓

Le niveau augmente progressivement la difficulté.

Le jeu n'a jamais eu besoin d'expliquer la règle avec du texte.

---

## Progression pédagogique

Une bonne introduction suit souvent quatre étapes.

### 1. Découverte

Le joueur observe une nouvelle mécanique.

---

### 2. Expérimentation

Il peut l'utiliser sans risque.

---

### 3. Maîtrise

Le défi augmente légèrement.

---

### 4. Combinaison

La nouvelle mécanique est combinée avec les précédentes.

Le joueur développe ainsi des compétences durables.

---

# 5.9 Les chemins

Un niveau peut proposer différents types de parcours.

## Chemin linéaire

Un seul itinéraire.

Avantages :

- rythme maîtrisé ;
- narration simple.

Inconvénients :

- faible rejouabilité.

---

## Chemins multiples

Le joueur choisit sa route.

Avantages :

- exploration ;
- liberté ;
- stratégies variées.

Inconvénients :

- conception plus complexe.

---

## Boucles

Le niveau revient vers une zone connue.

Cela permet :

- d'ouvrir des raccourcis ;
- de créer une cohérence spatiale ;
- de faciliter l'orientation.

Les jeux de type *Metroidvania* utilisent fréquemment cette structure.

---

# 5.10 Les points d'intérêt

Un point d'intérêt attire naturellement le joueur.

Exemples :

- un coffre ;
- une statue ;
- une fumée au loin ;
- un bâtiment en ruine ;
- une lumière inhabituelle.

Ces éléments encouragent l'exploration.

Ils servent également de récompense visuelle.

---

# 5.11 Le placement des ennemis

Les ennemis ne doivent jamais être placés au hasard.

Chaque groupe répond à une intention.

Exemples :

- enseigner une nouvelle mécanique ;
- contrôler la vitesse du joueur ;
- protéger une récompense ;
- créer une embuscade.

La composition des groupes est souvent plus importante que leur nombre.

---

# 5.12 Les récompenses cachées

Les niveaux gagnent en intérêt lorsqu'ils récompensent la curiosité.

Exemples :

- passages secrets ;
- coffres ;
- objets rares ;
- raccourcis ;
- éléments narratifs.

Le joueur apprend ainsi que l'exploration est bénéfique.

---

# 5.13 Greyboxing

Le **Greyboxing** consiste à construire un niveau avec des formes simples.

Le but n'est pas de créer un décor.

Le but est de tester :

- les distances ;
- les sauts ;
- la visibilité ;
- les combats ;
- le rythme.

La plupart des studios valident un niveau en greybox avant de produire les graphismes définitifs.

---

# 5.14 Étude de cas : Super Mario Bros. 1-1

Le premier niveau de **Super Mario Bros.** est souvent considéré comme un modèle de Level Design.

Pourquoi ?

- Le joueur avance naturellement vers la droite.
- Le premier Goomba enseigne qu'un ennemi est dangereux.
- Le bloc "?" apprend que les objets peuvent être frappés.
- Les tuyaux introduisent progressivement des obstacles plus hauts.
- Les fosses apparaissent après que le joueur a appris à sauter.

Le niveau enseigne toutes les mécaniques essentielles sans afficher un seul tutoriel textuel.

---

# 5.15 Erreurs fréquentes

## Trop d'informations

Multiplier les ennemis, les effets visuels et les objets interactifs rend le niveau difficile à lire.

Le joueur ne sait plus où regarder.

---

## Mauvais guidage

Si plusieurs chemins semblent identiques, le joueur peut se perdre.

Utilisez des repères visuels pour orienter son regard.

---

## Rythme monotone

Un niveau composé uniquement de combats ou uniquement de puzzles devient rapidement répétitif.

Variez les situations.

---

## Récompenses invisibles

Une récompense totalement cachée peut frustrer.

Le joueur doit avoir au moins un indice lui permettant de la découvrir.

---

## Décors trompeurs

Si un élément ressemble à une plateforme mais ne peut pas être utilisé, le joueur risque de perdre confiance dans les règles du jeu.

La cohérence visuelle est essentielle.

---

# Exercices

## Exercice 1

Dessinez sur papier un niveau de plateforme de 2 minutes.

Le niveau doit :

- enseigner une nouvelle mécanique ;
- comporter un défi principal ;
- inclure une récompense cachée ;
- se terminer par une zone calme.

---

## Exercice 2

Analysez un niveau d'un jeu que vous appréciez.

Identifiez :

- les points d'intérêt ;
- les chemins principaux ;
- les chemins secondaires ;
- les moments de tension ;
- les moments de respiration.

---

## Exercice 3

Réalisez une greybox.

Avec des cubes ou des formes simples, construisez un niveau en vous concentrant uniquement sur :

- les volumes ;
- les déplacements ;
- les lignes de vue ;
- les interactions.

Ne travaillez pas les graphismes.

---

# À retenir

- Le Level Design consiste à concevoir l'expérience spatiale du joueur.
- Un bon niveau guide naturellement sans multiplier les indications.
- La lisibilité est plus importante que la richesse visuelle.
- Le rythme alterne défis, exploration et moments de repos.
- Les nouvelles mécaniques doivent être enseignées progressivement.
- Le greyboxing permet de tester le gameplay avant la production artistique.
- Un excellent niveau raconte une histoire par sa structure autant que par son décor.
